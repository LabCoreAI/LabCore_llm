# Security Policy

## Supported Versions

Security updates are currently provided for:

| Version | Supported |
|---|---|
| 0.2.x | Yes |
| < 0.2.0 | No |

## Reporting a Vulnerability

Please do not open a public issue for security vulnerabilities.

Use private disclosure:

- GitHub Security Advisories (preferred), or
- Private contact with maintainers via GitHub.

Include:

- A clear description of the issue
- Impact and affected components
- Reproduction steps or proof of concept
- Suggested mitigation if available

## Response Process

- Initial acknowledgment target: within 7 days
- Triage and severity assessment
- Fix development and validation
- Coordinated disclosure once a fix is available
